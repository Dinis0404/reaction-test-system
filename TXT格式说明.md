# TXT 题目格式说明

## 支持的格式

系统支持两种题目文件格式：
- **TXT 格式**：纯文本格式，易于编辑
- **JSON 格式**：结构化格式，适合程序生成

## TXT 格式示例

每道题目之间用 `---` 分隔（或两个空行）：

```
题目1：JavaScript 中哪个方法用于向数组末尾添加元素？
A. push()
B. pop()
C. shift()
D. unshift()
答案：A
解析：push() 方法用于向数组末尾添加一个或多个元素。

---

题目2：CSS 中哪个属性用于设置元素的外边距？
A. padding
B. margin
C. border
D. spacing
答案：B
解析：margin 属性用于设置元素的外边距，即元素与周围元素之间的空间。
```

## 格式要求

1. **题目编号**（可选）：
   - `题目1：` 或 `题目1：` 开头
   - 也可以直接写题目内容

2. **选项格式**：
   - 使用 `A.`、`B.`、`C.`、`D.` 开头
   - 每个选项一行
   - 至少需要 2 个选项

3. **答案格式**：
   - `答案：A` 或 `答案：B` 等
   - 必须是大写字母 A、B、C、D
   - 答案必须对应其中一个选项

4. **解析**（可选）：
   - `解析：` 开头，后面跟解析内容
   - 如果没有解析，会显示"暂无解析"

5. **题目分隔**：
   - 使用 `---`（三个减号，可前后有空行）
   - 或两个连续空行

## 完整示例

```txt
题目1：React 中用于更新组件状态的 Hook 是什么？
A. useEffect
B. useState
C. useContext
D. useReducer
答案：B
解析：useState 是 React 中用于在函数组件中添加状态管理的 Hook。

---

HTML5 中哪个标签用于定义文档的主要内容区域？
A. <div>
B. <section>
C. <main>
D. <article>
答案：C
解析：<main> 标签用于定义文档的主要内容区域，每个页面应该只有一个 main 元素。
```

## 注意事项

- 文件名必须是 `.txt` 扩展名
- 文件放在 `data/` 目录下
- 选项可以使用 A-Z（但建议不超过 4-5 个选项）
- 答案索引从 0 开始（A=0, B=1, C=2, D=3）
- 支持中文和英文

## JSON 格式（参考）

如果使用 JSON 格式，格式如下：

```json
[
  {
    "id": 1,
    "question": "题目内容",
    "choices": ["选项A", "选项B", "选项C", "选项D"],
    "answerIndex": 0,
    "explanation": "答案解析"
  }
]
```
